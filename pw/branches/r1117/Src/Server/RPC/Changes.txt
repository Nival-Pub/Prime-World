Remote Objects (02.11.2009)
- Изменена кодогенерация: ID класса генерируется автоматически, его не нужно указывать в оригинальном классе. Все обертки необходимо
  перегенерировать, это можно сделать используя флаг --generate в при генерации файла проекта. Заходим в каталог с .appliction, 
  удаляем .auto\.cache, запускаем run <application> --generate и все обертки перегенерированы
- убран typename T в template <typename T, typename H, typename P> void Query(const char* path, H* object, void (H::func)(P result), const char password=0) 
- Добавлены различные информационные сообщения NI_ASSERT, NI_VERIFY, NI_ALWAYS_ASSERT, любой их них означает потенциально опасную ситуацию
- Разборка параметров переживает если в конструкторе были извлечены не все параметры
- Поддержаны со стороны темплейтов 7 параметров для методов
- Добавлена поддержка wstring  
- локальные классы можно передавать в качестве указателя через использование функции RemotePtr
- поддержано возвращение удаленных объектов
- удаленный указатель можно передавать в конструктор
- добавлен ассерт если сущность уже реплицирована на указанный Pipe
- поддержано возвращение локальных объектов из функций объявленных как REMOTE, опять же через RemotePtr
- тесты отсортированы по группам
- проведен рефакторинг по унификации передачи указателей удаленных и локальных объектов через маршаллинг
- поддержаны FixedVector скаляров только для конструкторов (например PopFixedVector<int, 20>) (осталось доработать кодогенератор)
